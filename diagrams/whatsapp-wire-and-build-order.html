You build this — the assignment

Your adapter · on_message()

glc/channels/catalogue/whatsapp/adapter.py — translates WhatsApp's dialect into the one shape the system understands. Four jobs, in this order:

  1. 1Verify the signature. Recompute the HMAC from the app secret. Unsigned or tampered request → return nothing. No envelope is ever built from an unverified body.
  2. 2Parse the payload. Meta: dig out entry[0].changes[0].value.messages[0]. Twilio: read form fields like WaId, Body.
  3. 3Classify trust. classify("whatsapp", sender) → owner_paired / user_paired / untrusted. Every message carries this stamp — it's what the policy engine reads later.
  4. 4Gate public channels. Stranger in a public context → silently dropped via allowed().

Output: a ChannelMessage — the typed Pydantic envelope. From here on, nothing downstream ever sees WhatsApp-specific JSON again.

Map 3 · Sir's widget, decoded

“One process, many channels, one viral arc”

The widget from the session page shows OpenClaw — the famous personal-agent project GLC learns from. Same layer-cake as Map 1, but the original, insecure version. Read each layer top-to-bottom: what it is, what broke there, and what GLC changed.

Widget layer 1 · “Agent loop”

LLM (propose) · Tools (dispatch) · Memory (read · write)

The brain: the LLM suggests an action, tools execute it, memory remembers between turns. This trio is every personal agent ever built.

What broke hereSummer Yue's inbox: the “ask before deleting” rule lived inside the LLM's chat context. Context filled up, got compacted, the rule vanished — 200 emails deleted, “STOP” ignored.
GLC's answerRules live in policy.yaml, a file outside the model. The LLM proposes; a deterministic policy engine authorises. “YAML does not compact.”

Widget layer 2 · The hub

OpenClaw Gateway — single long-running process, WebSocket control plane

One process on your computer connecting the brain to everything. GLC v1 is this box, rebuilt — that's why your repo is called a “gateway”.

What broke hereClawJacked: the WebSocket control plane trusted anything from localhost. A malicious webpage opened a connection, brute-forced a weak token, registered as a trusted device — and inherited every permission the agent had.
GLC's answerThe WS/control routes demand a per-install token; kill switch is out-of-band (/v1/control/kill); no generic shell endpoint — only named, typed handlers.

Widget layer 3 · Channels (fan-out) — your row!

WhatsApp · Telegram · Slack · Discord · Signal · Matrix · LINE · iMessage · WebUI · Voice

The same row your class is rebuilding as 22 group slots — WhatsApp, first box in the row, is yours. This fan-out is why the widget says “one process, many channels”: talk to the same agent from anywhere.

What broke hereMoltbook: agents posted into a social network whose Supabase DB was left wide open — 1.5M agent API tokens and 35K emails exposed. And the “1.5M agents” were really ~17K humans: the autonomy was theatre.
GLC's answerEvery envelope carries a trust_level; strangers are untrusted before any tool can run; every action lands in an append-only audit log the LLM can never write.

Widget layer 4 · Skill marketplace

browser-use · computer-use · calendar · payments · untrusted-skill

Loadable bundles that give the agent abilities — like installing apps. Note the widget literally includes a box named untrusted-skill.

What broke hereMalicious skills: 190 security advisories. A skill bundles instructions + scripts + API endpoints; users read the friendly markdown and missed the part quietly uploading their data to someone's server.
GLC's answerSkills don't get host execution; every tool call passes the policy engine + trust check first. (Full sandboxing arrives in Session 12 / glc_v2.)

The star chart at the bottom of the widget is the project's life story — each pin is a chapter (click them on the course page):

  1. Oct '25Clawd — Peter Steinberger (PSPDFKit founder) starts a personal-agent experiment. Single user, single machine. (blue pin: product)
  2. Nov '25Public release — 8,000 GitHub stars in weeks; gateway + channels + skills structure attracts contributors.
  3. Dec '25Viral — Moltbook: agents “form a society”, Reuters covers it, stars rocket toward 100K. (purple pins)
  4. Dec–FebThe four incidents — Moltbook DB exposure, Summer Yue's inbox, ClawJacked, malicious skills. The red pins at the very top of the curve: peak fame and peak failure together.
  5. Feb '26Acquihire — the founder joins OpenAI; the curve flattens. (orange pin)
  6. todayReference architecture — the design everyone (including GLC) learns from. As the widget's footer says: “the architecture is what carried; the spectacle did not.”

Why sir shows you this: your adapter is graded on avoiding these exact mistakes as much as on working — signature checks, trust stamping, and the public-channel gate in Map 1 each answer one red box above.

Why rebuild something already merged?

Because of the one rule that decides your grade: “no useful commit from a member → that member gets zero.” The group's adapter is merged, but none of those commits are yours. Steps 3–4 make you actually understand it; step 6 puts a genuine improvement in the repo under your name.
